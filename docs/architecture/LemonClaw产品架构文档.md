## 1. 产品定位

### 1.1 一句话定位

**LemonClaw 是 HomiClaw 的个人版，在保留核心能力的基础上，增加长期记忆和学习能力，让 AI 越用越懂你。**

│  ├── 多 Agent 架构 - 每个 Agent 独立配置、独立会话              │
│  ├── 技能系统 - 内置技能 + MCP 协议扩展                         │

│  └── 安全可控 - 权限管理、操作审计                              │
│                                                                 │

## 2. 架构来源与关系

### 2.1 HomiClaw 与 OpenClaw 的关系

HomiClaw 本质上是在 OpenClaw 上套了一层壳，增加了桌面应用和配置化可视化，以及一些异常处理手段和配置。

```
OpenClaw（开源 AI 助手平台，357k+ Stars）
    │
    │  提供：Agent Runtime、Tool 系统、Gateway 控制平面、
    │        MCP 集成、会话管理、Plugin SDK（100+ 子模块导出）
    │
    └──→ HomiClaw（蚂蚁内部定制）
          │
          │  增加的壳：
          │  ├── 桌面应用和配置化可视化界面
          │  ├── 异常处理增强（错误分类、重试策略、降级方案）
          │  ├── 内部服务集成（语雀/钉钉/Dima 等内部 MCP Server）
          │  ├── 企业权限审批和审计日志
          │  ├── SSO 集成
          │  └── 配置管理（分层配置、热重载）
          │
          │  直接复用的 OpenClaw 能力：
          │  ├── Agent Runtime（Agent 配置、生命周期、会话隔离）
          │  ├── Tool/Bash 工具系统
          │  ├── MCP Client
          │  ├── 配置 Schema（TypeBox）
          │  └── 认证轮换机制
          │
          └──→ LemonClaw（个人版）
                │
                │  继承自 HomiClaw 的设计模式：
                │  ├── AgentConfig 结构
                │  ├── Skill Registry 机制
                │  ├── 会话管理（Session/Message）
                │  ├── 技能执行流程（参数验证→权限审批→执行→审计）
                │  └── 事件系统（Agent 生命周期事件）
                │
                │  LemonClaw 自己的：
                │  ├── Electron 桌面应用（非复用 OpenClaw Gateway）
                │  ├── 长期记忆系统（参考 Hermes）
                │  ├── 学习引擎（LemonClaw 原创）
                │  └── 上下文压缩（参考 Hermes）
```

> **注意**：HomiClaw 与 OpenClaw 的具体集成方式待确认（是 fork 修改还是引用模块），已记录在 `private-docs/questions/HomiClaw待确认问题.md`。

### 2.1.1 参考项目：RivonClaw 的套壳方式

RivonClaw 是一个将 OpenClaw 包装为桌面应用的参考项目，其套壳模式值得 LemonClaw 参考：

```
RivonClaw 套壳架构：
├── vendor/openclaw/          — OpenClaw 二进制文件（不修改源码）
├── GatewayLauncher           — 启动/停止/重启 OpenClaw Gateway 子进程
│   └── 指数退避重启策略
├── Plugin Extensions         — 通过 OpenClaw 钩子注入
│   ├── before_agent_start    — 注入策略、文件权限
│   └── before_tool_call      — 自定义工具、权限控制
├── 数据层                    — 独立的 SQLite + 系统密钥链
├── 配置注入                  — 写入 ~/.openclaw/gateway/config.yml
│   └── 从 Keychain 注入密钥到环境变量
└── UI 层                     — React Panel + REST API（动态端口 HTTP 服务）
```

**RivonClaw 套壳模式要点：**
- **零源码修改**：vendor 目录的 OpenClaw 源码从不修改，可直接同步上游更新
- **子进程管理**：OpenClaw Gateway 作为子进程运行，桌面应用负责生命周期管理
- **钩子注入**：通过 OpenClaw 的 Plugin SDK 钩子（`before_agent_start`、`before_tool_call`）注入自定义逻辑
- **独立数据层**：规则、制品、频道、权限等存在自己的 SQLite，不依赖 OpenClaw 的存储
- **配置桥接**：将用户配置翻译为 OpenClaw 的 `config.yml`，密钥从系统 Keychain 注入

> LemonClaw 当前方案是**独立实现**（继承设计模式而非直接复用 OpenClaw Runtime）。如果未来需要复用 OpenClaw Runtime，可参考 RivonClaw 的 vendor + subprocess + hooks 模式。

### 2.2 代码复用策略

| 模块 | 策略 | 来源 | 说明 |
|------|------|------|------|
| **Agent Runtime** | 继承设计模式 | HomiClaw → OpenClaw | Agent 配置、生命周期、会话隔离 |
| **Skill Registry** | 继承设计模式 | HomiClaw → OpenClaw | 技能注册、执行、权限管理 |
| **MCP Client** | 继承设计模式 | HomiClaw → OpenClaw | MCP 协议客户端 |
| **会话管理** | 继承设计模式 | HomiClaw → OpenClaw | 会话结构、消息格式、事件系统 |
| **记忆系统** | 全新实现 | 参考 Hermes | HomiClaw 没有长期记忆 |
| **学习引擎** | 全新实现 | 参考 Hermes + 原创 | HomiClaw 没有学习功能 |
| **上下文压缩** | 全新实现 | 参考 Hermes | 长对话必备 |
| **错误恢复** | 全新实现 | 参考 Hermes + OpenClaw | 错误分类 + 自动恢复 |
| **桌面应用** | 全新实现 | Electron + React | HomiClaw 的桌面壳不适用 |
| **内部技能** | 不需要 | — | 不集成蚂蚁内部服务 |

## 4. 核心模块

### 4.1 Agent 系统（继承自 HomiClaw）

每个 Agent 独立配置、独立会话、独立记忆。

**包含内容：**
- Agent 配置（基础信息、模型配置、角色定义、技能配置、权限控制、记忆配置、学习配置）
- Agent 生命周期（创建→初始化→运行→等待→处理→响应→循环）
- Agent 事件系统（spawned/terminated/error/session:created/tool:called 等，用于 UI 状态同步）
- Session Key 命名空间（`agent:{agentId}:{sessionType}`，参考 ClawX）
- Snapshot 快照（CRUD 操作统一返回，确保前后端一致）

> 接口定义详见技术方案文档。

### 4.2 记忆系统（参考 Hermes + LemonClaw 增强 ⭐）

**四层记忆架构：**

| 层级 | 名称 | 存储 | 内容 | 管理方式 |
|------|------|------|------|---------|
| 第 1 层 | 会话上下文（短期） | 内存 | 当前会话消息历史 | 自动管理，超长压缩 |
| 第 2 层 | 会话搜索（跨会话） | SQLite + FTS5 | 过去会话记录 | FTS5 全文搜索 |
| 第 3 层 | 长期记忆（声明性） | MEMORY.md + USER.md + SQLite | 用户偏好、事实、事件 | 自动提取 + 可编辑 |
| 第 4 层 | 技能记忆（程序性） | SKILL.md 文件 | 可复用工作流 | 复杂任务后自动创建 |

**LemonClaw 增强点：**
- 信任评分（参考 Hermes Holographic，改为内置）— 不对称惩罚，宁可遗忘也不误导
- Agent 隔离记忆 — 每个 Agent 独立 MEMORY.md，互不干扰
- 冻结快照（参考 Hermes）— 会话开始时冻结记忆，保护 LLM 前缀缓存
- 上下文压缩（参考 Hermes）— 长对话自动结构化总结，防压缩风暴
- 安全扫描（参考 Hermes）— 写入前检测注入/外泄/不可见字符
- 记忆检索管线 — FTS5→信任加权→衰减调整→Top-K

> 具体接口（StructuredMemory、MemoryToolParams 等）、信任评分算法、检索管线、压缩算法详见技术方案文档。

### 4.3 学习引擎（LemonClaw 原创 ⭐）

Hermes 的学习是分散的（记忆/技能/反馈各自独立），LemonClaw 增加了集中式反思和统一学习报告。

**包含内容：**

| 组件 | 说明 | Hermes 有无 |
|------|------|------------|
| 经验收集器 | 自动收集用户修改、评分、纠正 | 无（被动收集） |
| 主动反思引擎 | 定期分析经验，生成学习报告 | 无 |
| 统一学习报告 | 可视化展示 Agent 学到了什么 | 无 |
| 技能即时修补 | 遇到问题立即修补 SKILL.md | 有（沿用） |
| 技能版本管理 | 技能更新支持回滚 | 无 |

**反思报告包含：**
- 统计概览（平均评分、修改率、满意度）
- 成功模式（用户满意的共同点 + 置信度）
- 发现的偏好（隐性偏好 + 证据）
- 需要改进（高频问题 + 建议修复）
- 建议操作（记忆更新、技能修补/创建）
- 所有建议需用户确认后才执行

> 反思报告完整接口定义详见技术方案文档。

### 4.4 技能系统（继承 HomiClaw + 参考 Hermes）

**技能分类：**

```
LemonClaw Skills
├── 内置技能 (Built-in) — 继承 HomiClaw
│   └── read / write / edit / exec / web_search
├── MCP 技能 (MCP Servers) — 继承 HomiClaw
│   └── filesystem / github / sqlite / 其他 MCP 服务
├── 自定义技能 (Custom)
│   └── 用户自定义
└── 学习生成技能 (Learned) ⭐ — LemonClaw 新增
    └── 反思引擎自动创建/修补
```

**技能执行流程（继承 HomiClaw）：**
1. 参数验证 → 2. 权限审批（safe/moderate/dangerous）→ 3. 安全扫描 → 4. 执行

**LemonClaw 增加（参考 Hermes）：**
- 技能文件 SKILL.md 格式（YAML frontmatter + Markdown）
- 安全扫描（数据泄露/提示注入/破坏性命令检测，结果分 SAFE/CAUTION/DANGEROUS）
- 渐进式加载（三级：列表→详情→文件内容，节省 token）
- 条件激活（`requires_toolsets` / `fallback_for_toolsets`，按场景动态加载）

> SkillRegistry 接口、Skill 接口、SKILL.md 格式详见技术方案文档。

### 4.5 LLM 调用层

- 多 Provider 支持（Theta GLM-5.1 / OpenAI / Claude / 本地模型）
- 流式响应
- 错误分类与恢复（参考 Hermes + OpenClaw）：限流→退避重试，认证失败→刷新重试，上下文溢出→压缩重试，计费耗尽→切换 Provider
- API Key 密钥链安全存储（macOS Keychain / Windows DPAPI）

> LLMProvider、ChatParams、错误分类枚举详见技术方案文档。

### 4.6 会话管理（继承 HomiClaw）

- 会话隔离（每个 Agent 独立会话）
- 历史持久化（SQLite）
- 事件系统（session:created / session:closed / tool:called 等）

### 4.7 配置管理

五层配置优先级（继承 HomiClaw + 参考 OpenClaw）：

```
1. 硬编码默认值 → 2. SQLite settings → 3. Markdown 文件 → 4. .env 文件 → 5. 系统密钥链
```

## 6. 竞品分析

### 6.1 与 HomiClaw 对比

| 特性 | **HomiClaw** | **LemonClaw** | 说明 |
|------|-------------|--------------|------|
| **定位** | 企业内部 | 个人使用 | 不同用户群体 |
| **与 OpenClaw 关系** | 套壳 + 增强 | 继承设计模式 | HomiClaw 直接复用 OpenClaw Runtime |
| **部署** | 内网服务 | 本地桌面应用 | 不同部署方式 |
| **开源** | 闭源 | 完全开源 | 可访问性 |
| **长期记忆** | 基础会话记忆 | **增强版** ⭐ | LemonClaw 创新 |
| **学习引擎** | 无 | **主动反思** ⭐ | LemonClaw 创新 |
| **上下文压缩** | 无 | **有** ⭐ | 参考 Hermes |
| **内部集成** | 语雀/钉钉等 | 不需要 | 使用场景 |
| **配置复杂度** | 高 | 低 | 用户体验 |

### 6.2 与 Hermes Agent 对比

| 特性 | **Hermes** | **LemonClaw** | 说明 |
|------|-----------|--------------|------|
| **记忆架构** | 分层 + 插件化 | 分层 + 内置 | 不需要配置外部插件 |
| **信任评分** | 仅 Holographic 插件 | 内置 | 不对称惩罚 |
| **冻结快照** | 有 | 沿用 | 保护前缀缓存 |
| **上下文压缩** | 有 | 沿用 | 防对话膨胀 |
| **安全扫描** | 有 | 沿用 | 防注入/外泄 |
| **主动反思** | 无 | **有** ⭐ | 定期分析经验 |
| **学习报告** | 无统一报告 | **有** ⭐ | 可视化学习成果 |
| **Agent 隔离** | 无（全局记忆） | **有** ⭐ | 每个 Agent 独立记忆 |
| **多 Agent** | 单 Agent | **多 Agent** ⭐ | 继承 HomiClaw |

### 6.3 全面对比

| 特性 | **LemonClaw** | **Hermes** | **OpenClaw** | **RivonClaw** | **Claude Desktop** |
|------|--------------|-----------|-------------|---------------|-------------------|
| **多 Agent** | 领域隔离 | 单 Agent | 单 Agent | 单 Agent | 单 Agent |
| **长期记忆** | 分层 + 内置 | 分层 + 插件 | 基础 | 无 | 有 |
| **信任评分** | 内置 | 需插件 | 无 | 无 | 无 |
| **主动反思** | 定期反思 | 无 | 无 | 无 | 无 |

| **上下文压缩** | 结构化总结 | 结构化总结 | 无 | 无 | 无 |
| **技能扩展** | MCP | MCP + 插件 | Tools | Plugin Hooks | MCP |
| **安全扫描** | 有 | 有 | 无 | 无 | 无 |
| **本地部署** | 完全本地 | 完全本地 | 部分 | 完全本地 | 云端 |
| **开源** | 完全开源 | 开源 | 开源 | 开源 | 闭源 |

## 7. 开发路线

### Phase 1：基础框架（继承 HomiClaw）（4 周）

**目标**：基于 HomiClaw 代码，搭建桌面应用骨架

```
Week 1-2: 桌面应用骨架
├── Electron 主进程（窗口 + 托盘 + 单实例锁）
├── React + shadcn/ui + Tailwind 前端
├── IPC 通信层（host-api + preload + ipc-handlers）
└── 基础页面路由

Week 3-4: Agent + 对话（复用 HomiClaw）
├── Agent Manager
├── LLMService（OpenAI SDK → Theta GLM-5.1）
├── 聊天界面（流式响应 + Markdown 渲染）
└── API Key 配置 + SQLite 存储
```

### Phase 2：记忆系统（参考 Hermes）（3 周）

**目标**：实现长期记忆系统

```
Week 5-6: 记忆引擎
├── SQLite（WAL 模式 + FTS5）
├── MEMORY.md / USER.md 读写
├── 信任评分 + 时间衰减
├── 记忆检索管线
├── 冻结快照 + 安全扫描
└── 上下文压缩

Week 7: 记忆 UI
├── 记忆列表 + 搜索 + 管理
└── 记忆使用状态面板
```

### Phase 3：技能系统（继承 HomiClaw）（3 周）

**目标**：复用 HomiClaw 技能系统 + MCP 集成

```
Week 8-9: Skill Registry（复用 HomiClaw）
├── 技能注册 + 执行 + 权限管理
├── 内置技能 + 安全扫描
└── 技能条件激活

Week 10: MCP 集成
├── MCP Client + MCP 服务器连接 + 工具调用
```

### Phase 4：学习引擎（LemonClaw 原创）（3 周）

**目标**：经验收集 + 主动反思

```
Week 11: 经验收集 + 技能即时修补
Week 12: 反思引擎（触发器 + LLM 分析 + 报告生成）
Week 13: 学习报告 UI + 技能版本管理
```

### Phase 5：优化 + 发布（2 周）

```
Week 14: 错误恢复 + 数据库优化 + UI 优化
Week 15: 文档 + 打包（Windows exe + macOS dmg）+ 发布
```

## 8. MVP 定义

### 8.1 MVP 功能清单（Phase 1 + Phase 2 部分）

| 优先级 | 功能 | Phase |
|--------|------|-------|
| **P0** | 单 Agent 对话 | 1 |
| **P0** | API Key 配置（密钥链存储） | 1 |
| **P0** | 对话历史保存（SQLite） | 1 |
| **P0** | 基础记忆存储（MEMORY.md + USER.md） | 2 |
| **P1** | 多 Agent 切换（2 个预设） | 1 |
| **P1** | 记忆检索（FTS5 关键词搜索） | 2 |
| **P1** | 3 个内置技能（文件读写、网页搜索） | 3 |

### 8.2 MVP 验收标准

- [ ] 用户可以与 Agent 进行多轮对话
- [ ] 对话历史被保存到本地
- [ ] 用户可以配置 API Key
- [ ] 用户可以在 2 个 Agent 之间切换
- [ ] 对话中的重要信息被存储为记忆
- [ ] 用户可以查看和搜索记忆

### 8.3 MVP 舍弃功能

| 功能 | 放入阶段 | 原因 |
|------|---------|------|
| 向量语义检索 | V2 | FTS5 够用 |
| 上下文压缩 | Phase 5 | 先确保基础功能 |
| 学习引擎 | Phase 4 | 先让记忆跑通 |
| MCP 集成 | Phase 3 | 先用内置技能 |
| 多 Agent 并行 | Phase 3 | 先做单 Agent + 切换 |
