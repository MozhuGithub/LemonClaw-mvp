## 🐛 踩过的坑

### 坑 1：ELECTRON_RUN_AS_NODE=1（耗时最长）
**症状**：`require('electron')` 返回字符串路径而不是 Electron API（app 为 undefined）
**根因**：Claude Code 自身设置 `ELECTRON_RUN_AS_NODE=1` 环境变量，导致 Electron 以纯 Node.js 模式运行，内置模块不加载
**解决**：dev 脚本通过 `cross-env ELECTRON_RUN_AS_NODE= electron-vite dev` 清除该变量

### 坑 2：pnpm + Electron 模块冲突
**症状**：npm `electron` 包的 index.js 返回二进制路径，遮蔽了 Electron 内置模块
**解决**：换用 electron-vite，它通过 externalizeDepsPlugin 正确处理模块外部化

### 坑 3：package.json 的 type: module 冲突
**症状**：加了 "type": "module" 后，Electron 主进程 CJS 编译产物报 exports is not defined
**解决**：去掉 "type": "module"，PostCSS/Tailwind 配置改用 .mjs 扩展名

### 坑 4：Electron 二进制下载失败
**症状**：默认源下载极慢或超时
**解决**：使用国内镜像 `ELECTRON_MIRROR=https://cdn.npmmirror.com/binaries/electron/`

## 💡 经验总结

1. **ELECTRON_RUN_AS_NODE** 是 Claude Code 环境的特殊变量，开发 Electron 时必须清除
2. **electron-vite** 是目前最成熟的 Electron + Vite 集成方案，不要手动配 Vite + Electron
3. **国内镜像**：`https://cdn.npmmirror.com/binaries/electron/` 下载 Electron 二进制
4. **用户偏好**：常规操作直接执行不确认，仅关键操作才找用户确认
5. **工作模式**：Windows 晚间开发 push，Mac 白天 pull + 写提示词，通过内部工具转存

## 🍎 Mac 环境验证注意事项

白天在公司 Mac 上 pull 代码后，验证功能前注意以下几点：

### 环境准备
1. **Node.js**：需要 >= 24.x（推荐用 nvm 安装：`nvm install 24 && nvm use 24`）
2. **pnpm**：需要 >= 10.x（`npm install -g pnpm`）
3. 如果 Mac 上没有 VS Build Tools，better-sqlite3 编译可能需要 Xcode Command Line Tools：`xcode-select --install`

### 启动步骤
```bash
git clone https://github.com/MozhuGithub/LemonClaw-mvp.git
cd LemonClaw-mvp
pnpm install                    # better-sqlite3 会为 Mac 重新编译
pnpm dev                        # 启动开发模式
```

### 需要验证的功能
- [ ] 窗口正常弹出，标题显示 "LemonClaw"
- [ ] 页面显示 "LemonClaw v0.1.0" 和平台信息（platform 应为 `darwin`）
- [ ] 点击 "Ping Main Process" 按钮，返回 "pong from LemonClaw main process"
- [ ] `window.lemonclaw.getInfo()` 在 DevTools Console 中能正常调用

### 已知兼容性
- Electron 32 支持 macOS 10.15+ (Catalina 及以上)
- Apple Silicon (M1/M2/M3) 和 Intel Mac 都支持（electron-vite 会下载对应架构的二进制）
- Mac 上不需要 `cross-env`，直接 `ELECTRON_RUN_AS_NODE= electron-vite dev` 也可以
- 如果在 Claude Code 环境下运行，同样需要清除 `ELECTRON_RUN_AS_NODE` 环境变量（dev 脚本已处理）

### 如果 pnpm install 报错
- Electron 二进制下载慢：设置镜像 `export ELECTRON_MIRROR=https://cdn.npmmirror.com/binaries/electron/`
- better-sqlite3 编译失败：确保已安装 Xcode Command Line Tools
- 权限问题：不要用 `sudo pnpm install`
