## 完成的工作

### 1. Step 3：基于源码通读修正实现

通读了 OpenClaw、HomiClaw、RivonClaw 三个参考项目的源码，修正了初始实现中的多个问题：

- **RPC 握手协议**：token 认证不含 nonce（nonce 仅用于 Ed25519 设备认证），client.id 必须为 `'openclaw-control-ui'`
- **Node.js 版本**：Electron 32 内置 Node v20.x 不满足 OpenClaw 要求的 v22.12+，改用系统 Node
- **Vendor 构建**：submodule 需 `pnpm build` 生成 `dist/entry.js`

### 5. Config Bridge 重大修正

对照用户本地可用的 openclaw.json，修正了配置格式：

## 关键踩坑

| 问题 | 根因 | 修复 |
|------|------|------|
| Gateway 立即退出无输出 | Electron Node v20.x < v22.12 | 系统Node.js |
| "unexpected property 'nonce'" | token 认证不需要 nonce | auth 只含 `{ token }` |
| "client/id must be constant" | client.id schema 验证 | 改为 `'openclaw-control-ui'` |
| "Missing config" | vendor 无全局配置 | OPENCLAW_CONFIG_PATH 环境变量 |
| "plugin path not found" | extensions 目录不存在 | existsSync 检查 |
| "missing scope: operator.write" | token 认证无设备身份，scope 被清 | `openclaw-control-ui` + `dangerouslyDisableDeviceAuth` |
| "origin not allowed" | Control UI 触发浏览器 origin 检查 | WebSocket 加 `origin` header |
| Gateway 反复重启 | plugins.allow 引用不存在的 `minimax` 插件 | 清除无效插件引用，改用 `entries` |
| Gateway "正在启动" 不转 running | launcher 过早设 running（stdout "listening" 时 RPC 未连） | 过滤 launcher 的 running 状态，等 RPC 连接后才通知前端 |
| chat.send 无事件返回 | models.providers 缺 apiKey + auth-profiles provider 名不匹配 | 对照用户可用配置重写 config-bridge |
