## 🐛 踩过的坑

### 坑 1：tailwind.config.mjs 不兼容 tailwindcss-animate
**症状**：`require is not defined` 报错
**根因**：`.mjs` 强制 ESM，但 `tailwindcss-animate` 内部用 `require()`
**解决**：重命名为 `.js`，使用 `module.exports`

### 坑 2：shadcn-ui CLI 已废弃
**症状**：`npx shadcn-ui@latest add` 提示 deprecated
**解决**：改用 `npx shadcn@latest add`

### 坑 3：JPG 格式不适合做 Electron 图标
**症状**：用户提供的图标是 JPG 格式
**根因**：JPG 不支持透明通道，macOS 打包需要 PNG 源文件
**解决**：待用户提供 PNG 格式图片

### 坑 4：cross-env 在 Windows 上无法 unset 环境变量
**症状**：`cross-env ELECTRON_RUN_AS_NODE=` 在 Windows 上只设空值，不 unset
**根因**：Windows 没有真正的 unset 概念，cross-env 只能设空
**解决**：dev 脚本改用 `env -u ELECTRON_RUN_AS_NODE electron-vite dev`

## 💡 经验总结

1. **shadcn/ui 手动初始化**：electron-vite 的目录结构和标准 Vite 不同，CLI 向导会配错路径，手动创建 `components.json` + 配别名更可靠
2. **CJS vs ESM**：Tailwind 配置用 `.js`（CJS）更稳，很多插件还在用 `require()`
3. **图标格式**：Electron 图标必须 PNG（透明通道 + macOS 打包），JPG 不行
4. **图片格式问题找用户**：不擅自改格式，让用户导出正确格式
