## Phase 1: 基础框架

**目标**：能和 AI 正常对话的桌面应用（流式响应 + API Key 安全存储）

### Step 1: Electron 安全骨架 ✅

**做了什么**：搭建 Electron 桌面应用骨架，包含安全隔离、IPC 通信、React 前端

**有什么用**：后续所有功能的基础运行环境，确保主进程和渲染进程安全隔离

**状态**：✅ 完成（2026-04-16）

---

### Step 2: 前端框架搭建 ✅

**做了什么**：集成 shadcn/ui 组件库，建立暗色主题（zinc 色系 + lemon 品牌色），实现 Sidebar + PageRouter 布局，Zustand 状态驱动路由（无 react-router），安装 button/separator/scroll-area/tooltip/input/card 六个 shadcn 组件

**有什么用**：提供统一的 UI 组件和布局框架，后续所有界面在此基础上开发

**状态**：✅ 完成（2026-04-17）

---

### Step 3: IPC 通信层

**做什么**：建立 host-api 抽象层，前端组件通过统一接口调用主进程能力，不直接接触 IPC 细节

**有什么用**：解耦前后端，方便后续扩展 IPC 通道（聊天/Agent/配置等）

**状态**：⬜ 未开始

---

### Step 4: LLM 调用层

**做什么**：接入 Theta GLM-5.1 API，支持流式响应、错误分类和自动重试

**有什么用**：核心能力——让应用能真正调用 AI 模型生成回复

**状态**：⬜ 未开始

---

### Step 5: 单 Agent + 基础聊天

**做什么**：实现单 Agent 对话，包含消息气泡、流式渲染、聊天状态管理

**有什么用**：最小可用产品——用户能和 AI 对话

**状态**：⬜ 未开始

---

### Step 6: API Key 配置

**做什么**：API Key 安全存储（系统密钥链）+ 配置界面 + 多 Provider 支持

**有什么用**：用户可以自己配置 API Key，支持多个 AI 服务商

**状态**：⬜ 未开始

## 技术决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 构建工具 | electron-vite | 比 electron-forge 更成熟的 Vite 集成 |
| 包管理 | pnpm (node-linker=hoisted) | Electron 模块解析需要 |
| LLM 调用 | OpenAI SDK → Theta GLM-5.1 | MVP 不依赖 OpenClaw Gateway |
| 记忆方案 | Markdown + SQLite 混合（参考 Hermes） | 四层分层 + 信任评分 + 冻结快照 |
| 密钥存储 | electron safeStorage | 利用系统密钥链（Keychain/DPAPI），不存明文 |
| 去掉规则引擎 | 行为定制用 systemPrompt + 记忆 | 对个人用户不必要，规则引擎是企业场景 |
| 继承设计模式 | 独立 TypeScript 实现，不复用 OpenClaw 代码 | 保持独立性，记忆/学习引擎需自由设计 |
| UI 组件库 | shadcn/ui + Tailwind | 暗色主题友好、可定制、中文生态好 |
| 页面路由 | Zustand 状态驱动 | 桌面应用无需 URL 路由，简单够用 |
| 备选复用方案 | RivonClaw 的 vendor + hooks 模式 | 如果 Phase 3+ 复杂度超预期可切换 |
